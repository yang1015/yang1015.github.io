hexo + github 史上最简单的blog搭建教程

Posted on 2018-01-04

想起两年前刚入门编程的时候，就想自己搞一个技术博客来记录自己的成长，奈何那时候零基础什么都不懂，人家教程上的代码都放在那里了我都不知道该ctrl+v到哪里才能起效。 转眼两年，工作也已经8个月了。一直有写技术笔记的习惯，想着18年想把自己的一些学习心得分享出去，也能让更多人来纠我的毛病，于是想着再来搜一回教程把博客搭起来。23333

参考了一些前辈们的教程，然后在实际搭建中又遇到了更多的坑，希望这次能替后来的小白如我填更多一些坑吧。
（于是这即将成为一篇特别啰嗦的教程 希望啥都不会的小白也能跟下来并且成功搭好博客 ╭(╯^╰)╮）

教程开始

开启本地新世界

1- 安装node

2- 安装git

3- 安装hexo

官网地址: Hexo官网地址 戳
安装指令:

$ sudo npm install -g hexo
注意: 这里一定要带上sudo，否则会因为权限问题报错

4- 创建本地的blog项目

在你想建立这个项目的路径里，初始化你的blog项目。 举个栗子 比如我mac里有个文件夹是我私人的文件夹叫skylar, 我想在skylar这个文件夹里微博我的blog项目，那么就先cd切到skylar这个文件夹下
cd documents/skylar
然后呢在skylar文件夹里初始化你想建立的blog

hexo init blog
这里的blog就是你自己起的项目名儿 这个无所谓的 name it whatever you want 接着 切到blog这个文件夹下 安装依赖包

npm install
最后 起服务器 command line将告诉你listen的端口号

hexo s
本地地址为: http://localhost:4000
直接在浏览器里打开下面这个本地就好了
直接戳这里

然后咧 你就会看到你本地的blog创建成功了 👇

啪啪啪啪啪啪啪👏 好了第一阶段是不是非常简单 本地起好了 但是别人看不到呀
为了让我们的blog不是一个单机游戏 为了能够打squad四排为了苟到最后成功吃鸡！！(雾) 我们接下来要干的事儿就是把它布出去，跟github捆绑销售，让队友们都一起上飞机一起跳伞一起落地成盒！！(再次大雾)

队友github上线 一起吃鸡一起飞啊22333

1- 登录你的github账户

没有github的账户的先去注册一个 作为技术工种的我们 没有github都没脸说自己是技工 (虽然我也才用没多久 打脸)
有账号的同学 来来来 我继续说 首先 先创建一个新的repository 然后把repository name规定为[你的账户名] + github.io
举栗：
我github account是yang1015 那么我要创建的这个仓库名就是yang1015.github.io

这样仓库就建好了

接着我们cd到你所创建的blog项目下 以我自己的文件夹举例子：

cd documents/skylar/blog
cd blog
ls
blog文件夹下有一个名为_config.yml的文件，我们现在要做的就是编辑这个文件里的内容 使用vim指令来编辑

今天查搭建博客的教程的时候 我也是第一次用vim 憋怕 我把这里会用到的vim指令全告诉你XD

// 开始之后是注释 不要复制啊 我知道我是白说 = = 但是怕有小白嘛

vim _config.yml // 预览这个文件里的所有内容
/
此时有超级多的内容 不管 直接拉倒最末尾 寻找关键字”deploy” 我们要将它本来的deploy配置改成可以自动连接我们本地博客和github的那种神奇配置 /

i // 该指令是将命令行模式切换到编辑模式 输入i之后你才能开始修改该文件的内容
把你的光标移到需要编辑的地方 把它原本的配置改成这样 👇

type: git
repo: https://github.com/xxx/xxx.github.io.git
branch: master
xxx为个人github的name
比如我的话就应该改成: yang1015/yang1015.github.io.git

注意注意注意重要的事情说三遍！！！

在修改配置的时候 此处有神坑 因为yml这个特殊的格式是有特殊的要求的，它要求你每一个冒号后面都要先有一个空格 然后再来你想写的值
比如type:git ×

但是type: git √
不听话的话会报错的 比如报那个indentation什么什么的错
2- 配置完毕 生成静态页面

在blog路径下
hexo generate // 或者简写为 hexo g，这里是为了生成一个静态页面
此时新坑又来了 如果这里报错的话 那么挽救一下 执行以下command(没报错的跳过这一段代码)

npm install hexo –save
紧接着

hexo deploy // 或者简写为 hexo d，执行配置
然后天降又一神坑 这坑我就遇上了(没有遇到以下error的跳过这一段)

什么什么deployer not found之类的error
那么我们来执行以下命令

npm install hexo-deployer-git –save
最后再次执行一遍：

hexo g
hexo d
以上代码执行成功之后 点这里 看你的博客布上去了没
没问题了是吧 网址能打开吧 里面是刚才你本地localhost 4000看到的那个博客样子吧？
